# 000000 000 0 000000 춘계학술대회 프로그램북 자동 생성

000000 000 0 00000 **2026년 춘계학술대회** (2026-mm-dd~dd) 프로그램북을
`input_raw.xlsx`(학회 submission 원본) → `program_book_working.xlsx`(다중 시트 통합 작업본)로
자동 가공하는 Python 파이프라인.

**설계 철학**: 완전 자동 배치는 비현실적(특별세션·학생경진대회·발표장 전통·추가 발표자 등 변수 과다).
따라서 **"포맷팅 도우미"** 접근 — 사용자가 엑셀에서 논문별 배정만 채우면, Python이
(a) 미배정 검출, (b) 충돌 검증, (c) 소속 축약, (d) 매트릭스/세부일정 시트를 자동 생성.

---

## 1. 파일 구조

```
auto_scheduling/
├── generate_program.py         # CLI 엔트리
├── input_raw.xlsx              # 학회 원본 (read-only)
├── program_book_working.xlsx   # 단일 작업 파일 (모든 시트가 이 안에)
├── HANDOVER.md                 # 초기 설계 노트 (과거 스냅샷, 일부 컬럼명 구버전)
├── README.md                   # (이 문서)
└── scripts/
    ├── __init__.py
    ├── step1_cleaning.py       # 정제
    ├── step2_lab_mapping.py    # 랩/소속 정규화
    ├── step3_flagging.py       # 제약조건 플래그 + 좌장/Invited 템플릿
    ├── step4_assignment.py     # 배정 검증 + 설정_시간대/입력_논문배정 템플릿
    └── step6_formatting.py     # 최종 출력 시트 생성
```

Step 5는 없음(검증 기능이 Step 4에 통합됨 — 번호는 작업 이력 흔적). 필요 패키지: `pandas`, `openpyxl`.

## 2. CLI 사용법

```bash
python generate_program.py --init          # 최초 1회: input_raw.xlsx → 시트2로 복사해서 작업 파일 생성
python generate_program.py --step 1        # 정제
python generate_program.py --step 2        # 랩 매핑
python generate_program.py --step 3        # 플래그 + 좌장/Invited 템플릿
python generate_program.py --step 4        # 설정_시간대/입력_논문배정 템플릿 + 배정 검증
python generate_program.py --step 6        # 구두/포스터 명단, 시간표 매트릭스, 세부일정 생성
python generate_program.py --step all      # 1~6 순차 실행
```

반복 작업 흐름:
1. 엑셀에서 `입력_논문배정`에 `날짜/세션/발표장/슬롯순서/슬롯길이` 채우고 저장/닫기
2. `--step 4` → `_step4_배정현황`, `_step4_충돌경고`로 검증
3. 문제 있으면 1로 복귀, 없으면 `--step 6`으로 최종 시트 생성
4. 한글(HWP)에 복붙

## 3. `program_book_working.xlsx` 시트 전수 목록

시트는 크게 3그룹으로 분류됨.

### ① 사용자 편집 시트 (재실행해도 **절대 덮어쓰지 않음**)

| 시트 | 생성자 | 역할 |
|---|---|---|
| `시트2` | `--init` | 원본 submission. 발표자/소속/제목/등록여부 등 44컬럼 |
| `입력_좌장Invited` | Step 3 (최초 1회) | `Invited` 컬럼에 `Y` 마킹 → Step 6이 매트릭스/세부일정에 `[Invited]` / `Invited Paper*` 자동 삽입 |
| `설정_시간대` | Step 4 (최초 1회) | 날짜·15분 슬롯·세션 매핑 (하단 상세) |
| `입력_논문배정` | Step 4 (최초 1회, 이후 참고컬럼은 refresh) | 사용자가 논문별 `날짜/세션/발표장/슬롯순서/슬롯길이` 기입 |
| `입력_세션좌장` | Step 6 (최초 1회) | 세션별 `세션명/좌장` 기입. 미입력 시 `TBD` |

보존 메커니즘: 각 Step이 `sheet_exists()` 헬퍼로 존재 여부를 확인하고, 있으면 템플릿 덮어쓰기를 건너뜀. 단 `입력_논문배정`의 **참고 컬럼**(`발표자/발표형식/발표분야/랩대표`)은 시트2 변경이 반영되도록 매번 refresh됨 — 사용자가 채운 **배정 컬럼**(`날짜/세션/발표장/슬롯순서/슬롯길이/비고`)은 건드리지 않음.

| `_step1_예외항목` | 1 | 제외된 논문 + 복구방법 |
| `_step2_랩매핑` | 2 | `논문번호 / 랩대표 / 랩대표_소속_원본 / 랩대표_소속_정규화 / lab_id` |
| `_step2_소속정규화` | 2 | `원본표기 → 정규화` 매핑표 |
| `_step2_랩요약` | 2 | 랩별 발표 건수 요약 |

| `포스터 발표 명단` | 포스터 발표자 리스트 (`번호/포스터코드/논문번호/세션/발표자/소속/직위/발표분야/제목/좌장`) |

## 4. 파이프라인 단계별 로직

### Step 1 — 정제 (`step1_cleaning.py`)

순서 의존성이 있는 7단계:
1. **제목 줄바꿈 병합**: 논문번호가 비어있는 행의 제목을 직전 논문에 append
2. **빈 행 제거**: 논문번호 없으면 drop
3. **마크다운 제거**: `**bold**`, `__italic__` 마커 제거
4. **중복 제출 제거**: `(발표자 + 제목)` 동일 시 **등록일 최신**만 유지
5. **미등록/미납 제외**: `사전등록여부≠등록완료` OR `사전등록 결제여부≠결제완료` → 예외항목으로 분리
6. **발표형식 메모 처리**: `구두발표(포스터로 수정필요)` → `포스터발표` 등
7. **발표분야 오타 교정**: `TYPO_MAP`으로 `[특별세선]` → `[특별세션]` 등

예외항목 컬럼: `논문번호, 발표자, 예외유형, 상세, 복구방법, 원본발표형식, 원본발표분야`

### Step 2 — 랩/소속 식별 (`step2_lab_mapping.py`)

**랩대표** 판정:
- 단독 저자 → 본인
- 발표자 직위가 시니어(`교수/책임연구원/수석연구원/실장/연구교수/대표/CEO/상무/이사/매니저`) → 발표자 본인
- 그 외(학생/연구원) → 저자 목록의 **마지막 사람** (보통 지도교수)

**소속 정규화** 순서:
1. 슬래시/쉼표 있으면 첫 번째 기관만 (`고등기술연구원 / KAIST` → `고등기술연구원`)
2. 영문 약칭 표준화 (`KAIST`→`한국과학기술원`, `GIST`, `UNIST`, `POSTECH/포스텍`→`포항공과대학교`)
3. 괄호 제거
4. `ORG_SUFFIXES`(`대학교/과학기술원/공과대학/대학원/연구원/연구소/기술원/대학`)로 접미 잘라냄

`lab_id = "{랩대표}__{정규화소속}"` (예: `홍길동__한양대학교`)

### Step 3 — 제약조건 플래그 (`step3_flagging.py`)

`_step3_플래그` 컬럼: `is_multi_presenter`, `multi_presenter_count`, `multi_peer_papers`, `is_special_session`, `is_chair_requested`.

**`입력_좌장Invited`** 템플릿(최초 1회): `논문번호, 발표자, 발표형식, 발표분야, Invited, 좌장여부, 참고사항`.
사용자가 `Invited` 컬럼에 `Y` 기입하면 Step 6 출력에 반영됨.

### Step 4 — 배정 검증 (`step4_assignment.py`)

**가장 중요한 Step**. 배치 알고리즘이 아닌 검증 도우미.

**`설정_시간대`** 스키마 (기본 날짜별 17슬롯 × 2일 = 34행):
```
날짜 | 시간대  | 시작  | 종료   | 세션
목   | 오전1~5 | 9:00  | 10:15 | 목-오전1
목   | 휴식1~2 | 10:15 | 10:45 | (비움)
목   | 오전8~12| 10:45 | 12:00 | 목-오전2
목   | 오후1~5 | 13:30 | 14:45 | 목-오후1
금   | (동일)  |       |       | 금-*
```
- 1개 세션 = 15분 × 5슬롯 = 75분
- 기본값엔 오전1·오전2·오후1 3개 세션만 있음. `오후2`(코드 `D`)는 세션명 suffix 매핑엔 있으나 기본 타임슬롯엔 없음 — 필요하면 사용자가 시트에 직접 추가
- `세션` 컬럼 값이 `입력_논문배정.세션` 드롭다운 소스

**`입력_논문배정`** 스키마(11컬럼):

| 컬럼 | 타입 | 설명 |
|---|---|---|
| 논문번호 | str | Primary key. `_step1_정제데이터` 기준 |
| 발표자 | str (refresh) | 참고용 — 매 Step 4 실행마다 덮어씀 |
| 발표형식 | str (refresh) | `구두발표` / `포스터발표` |
| 발표분야 | str (refresh) | `[일반]`, `[특별세션] ...`, `[신진연구자]...`, `[튜토리얼]` 등 |
| 랩대표 | str (refresh) | 중복 검출 시 동명이인 구분용 |
| **날짜** | `목`/`금` | 드롭다운 |
| **세션** | `목-오전1` 등 7종 | 드롭다운 (설정_시간대에서 유도) |
| **발표장** | `1`~`7` | 드롭다운, 문자열 |
| **슬롯순서** | `1~5` (구두) / `A01`, `B19` 등 (포스터) | 포스터는 letter+숫자, 정수 부분만 추출 |
| **슬롯길이** | `1~5` | 1개 기본, 초청강연은 4~5 |
| 비고 | 자유 입력 | |

**검증**:
- `VALID_DATES = {'목', '금'}`, `VALID_VENUES = {'1',…,'7'}`, `MAX_SLOT_POSITION = 5`, `MAX_SLOT_SPAN = 5`(구두만)
- 구두: `슬롯순서 + 슬롯길이 - 1 > 5` → 세션 경계 초과 오류
- `_step4_충돌경고` 유형:
  - `배정값_오류` — 날짜·세션·발표장·슬롯순서/길이 범위/형식 위반
  - `슬롯_중복점유` — `(날짜, 세션, 발표장, 슬롯)`에 구두 2건 이상 (슬롯길이 전개 포함)
  - `발표자_시간충돌` — 같은 `(발표자, 랩대표, 날짜, 세션)`에 구두 2건 이상. 랩대표가 다르면 동명이인으로 통과. 포스터는 검증 대상 아님

**Refresh·정리 로직** (최초 생성 이후 `--step 4` 재실행 시):
- 참고컬럼(`발표자/발표형식/발표분야/랩대표`)만 `_step1_정제데이터` 기준으로 갱신. 배정컬럼은 보존.
- `_step1_정제데이터`에 없는 고아 pid 행은 삭제 (취소된 발표 정리).
- 필수 배정값(`날짜/세션/발표장/슬롯순서`) 중 하나라도 빈 행은 `비고='학생경진대회만'` 자동 태깅.
- `_step1_정제데이터`에 있는 신규 pid는 템플릿 행으로 머지.

**`_step4_소속축약` 규칙**:
- 특수: `한국과학기술원→KAIST`, `울산과학기술원→UNIST`, `광주과학기술원→GIST`, `포항공과대학교→POSTECH`
- 공통: `XX여자대학교→XX여대`, `XX대학교→XX대`
- 그 외(연구원/회사): 원본 유지

### Step 6 — 최종 포맷팅 (`step6_formatting.py`)

**세션 코드 생성 (구두)**:
```
KSME 26CA-{Day}{Venue}{Slot}{Position}
```
- `Day`: `목→Th`, `금→Fr`
- `Venue`: 발표장 2자리 (`01`~`07`)
- `Slot`: `오전1→A`, `오전2→B`, `오후1→C`, `오후2→D` (세션명 suffix 기준)
- `Position`: `슬롯순서` 2자리

예: `KSME 26CA-Th01A01` = 목요일 1발표장 오전1세션 1번째.

**포스터 코드 생성**:
```
KSME 26CA-{Day}08{Slot}{Position}
```
구두와 구분하려고 venue 자리에 고정값 `08` 사용. `Position`은 사용자가 입력한 `슬롯순서`(`A14`/`C16` 등)에서 정수부만 뽑아 2자리로 패딩. 세션 레터는 `세션`값 기준(슬롯 접두어는 검증만 하고 코드에는 사용하지 않음).

예: `KSME 26CA-Th08A14` = 목요일 오전1세션 포스터 14번.

**포스터 사전 검증** (실패하면 Step 6 중단):
- `_validate_poster_field_contiguity` — 같은 `(날짜, 세션)` 내에서 `발표분야`가 한 번 끝난 뒤 다시 등장하면 오류. 포스터는 분야별로 연속 블록이어야 함.
- `_validate_poster_slot_numbers` — 같은 `(날짜, 세션)` 내 슬롯 번호 중복 차단 (`A16`과 `C16`이 섞여 있으면 코드 충돌).
- `_validate_poster_slot_prefix` — 슬롯 접두어(A/B/C/D)가 세션 레터와 일치해야 함.

**매트릭스 셀 포맷**:
```
{[Invited] }{발표자}
({소속축약})
```
매트릭스는 세션 블록마다 `세션명` 행과 `좌장` 행이 자동으로 삽입되며, 세션/휴식 블록이 바뀔 때 공백 행 1줄이 들어감.

**포스터 세션 배치 셀 포맷**:
```
{발표자}
({소속축약})
[{슬롯순서}]
```
행 구조: `포스터 세션 A (...)` 타이틀 행 → 분야 헤더 행(같은 분야 가로 병합) → 포스터 행. 14열 단위로 끊어서 반복. 타이틀은 주황(`F6C28B`), 분야별 배경색은 `style_poster_session_sheet()`의 `field_fills` 매핑.

**발표 세부 일정 한 줄 포맷** (2컬럼):
```
[세션코드] [{Invited Paper*  }{제목} / {저자_포맷}]
```
저자 포맷: 발표자에 `*` 부착, 각 저자에 `(소속축약)`. 예: `김우빈*(한양대), 홍길동(KAIST)`.
발표장 사이는 줄바꿈 없음, 같은 발표장 내 날짜 사이 2줄, 세션 사이 1줄.

**`입력_세션좌장`** 템플릿: `날짜 / 세션 / 발표장 / 발표장명 / 세션명 / 좌장 / 비고`.
`발표장명`은 `VENUE_NAMES`(그랜드볼륨I/II, 사파이어I/II/III, 릴리I/II)로 자동 채움. 세션명/좌장 미입력 시 `TBD` 표기.

**매트릭스 셀 병합**: 휴식 행은 가로병합, `슬롯길이>1`인 발표는 세로병합. 포스터 세션 배치 시트도 `apply_merges()`로 타이틀 행/분야 헤더 병합 처리.

## 5. 특수 발표(신진연구자·튜토리얼) 추가 방법

학회 submission에 없는 초청 발표(신진연구자, 튜토리얼 강연 등)는 **9000번대 합성 논문번호**로 `시트2`에 직접 추가:

```python
{
    '논문번호': '9001', '발표자': '홍길동', '회원구분': '학회원',
    '회원이름': '홍길동', '소속': '한양대학교', '부서': '기계공학과',
    '발표형식': '구두발표', '발표분야': '[신진연구자] 초청 발표',
    '제목': '신진연구자 초청 발표 (홍길동)',
    '저자정보-성명': '홍길동', '저자정보-소속기관': '한양대학교',
    '사전등록여부': '등록완료', '사전등록 결제여부': '결제완료',
    '등록일': '2026-04-01',
},
```

추가 후 `--step all` (또는 1→2→3→4)로 파이프라인 태우면 `입력_논문배정`에 자동 머지됨. 배정값은 사용자가 직접 기입하거나 별도 스크립트로 세팅.

튜토리얼처럼 50분(≈4슬롯) 짜리는 `슬롯길이: '4'`로 지정.

## 6. 유용한 헬퍼 (generate_program.py)

- `init_working_file()` — `input_raw.xlsx`의 첫 시트를 `시트2`로 복사
- `write_sheet(df, name, header=True)` — 있으면 replace, 없으면 신규. 매트릭스/세부일정 시트는 `header=False`
- `sheet_exists(name)` — 사용자 편집 시트 보존 체크용
- `apply_assignment_dropdowns(df_timeslots)` — `입력_논문배정`에 5컬럼 드롭다운 적용(`슬롯순서`는 포스터 letter 값 허용을 위해 `enforce=False`)
- `snapshot_sheet_cells(name)` / `restore_empty_cells(name, snap)` — 수동 셀 보존
- `apply_merges(name, merges)` — 매트릭스 병합 적용
- `autofit_columns(names)` — CJK 2칸 폭 기준으로 컬럼 너비 자동 조정
- `style_poster_session_sheet(name)` — 포스터 세션 배치 시트에 타이틀/분야 색상·테두리·중앙정렬 적용

## 7. 알려진 주의사항

1. **수동 셀 보존의 한계**: `5월7일(목)` 등에서 행/열 삽입으로 기존 셀 위치가 밀리면 스냅샷 좌표와 어긋남 → 복원이 이상한 곳에 일어남. 자동 생성된 그리드 형태는 유지하고, 값만 채우는 것이 안전. (`preserve_user_cells` = `{'5월7일(목)', '5월8일(금)'}`만 해당, 포스터/세부일정 시트는 덮어씀)
2. **포스터 슬롯순서**: `A01`, `B19`처럼 letter 접두어를 그대로 입력. 접두어는 세션 레터(`오전1→A` 등)와 반드시 일치해야 함(`_validate_poster_slot_prefix`). 숫자부만 `re.search(r'\d+')`로 추출해 코드 생성·정렬에 사용(갭 허용: `B19` 다음 `B35` 가능).
3. **포스터 분야 블록 연속성**: 같은 세션 내에서 `발표분야`가 한 번 끝나면 다시 등장 금지. Step 6가 중단됨 → `입력_논문배정`에서 같은 분야끼리 연속되게 `슬롯순서` 재조정 필요.
4. **포스터 슬롯 숫자 고유성**: 같은 세션 안에서 `A16`과 `C16`처럼 숫자가 겹치면 포스터 코드가 중복되므로 Step 6 차단.
5. **`_clean_cell()`**: pandas NaN과 문자열 `'nan'` 둘 다 빈 문자열로 정규화. Step 4에서 빼먹으면 `'nan'`이 유효값처럼 처리되는 버그 발생(과거 실제 발생).
6. **세부일정 시트 구조**: 2컬럼(`발 표 세 부 일 정`/`포 스 터 발 표 세 부 일 정`, 내용). `header=False` 필수(`headerless_sheets` set 참고).
7. **드롭다운 소스 변경 시**: `설정_시간대`의 `세션` 값을 고치면 `입력_논문배정` 기존 입력이 드롭다운 목록에서 벗어날 수 있음 → Step 4 재실행으로 드롭다운 재적용.
8. **고아 pid 자동 삭제**: Step 4 재실행 시 `_step1_정제데이터`에 없는 `입력_논문배정` 행은 삭제됨. 시트2에서 제외했다가 되살릴 경우 배정값이 사라지므로 주의.

## 9. 핵심 설계 결정 (사용자 협의 결과)

코드 전반에 박혀 있으므로 변경 시 영향 범위 확인 필수:

1. **단일 엑셀 + 다중 시트** (다중 파일 거부)
2. **랩대표는 직위 기반 판별** (학생=지도교수, 시니어=본인)
3. **슬래시 소속은 첫 기관 우선**
4. **영문 약칭은 4대 과기원만**(KAIST/UNIST/GIST/POSTECH), 나머지는 `대학교→대` 자동
5. **중복 제출은 최신 유지**
6. **미등록/미납은 예외항목에 복구방법 기재**
7. **배치 알고리즘 X, 검증 도우미 O**
8. **사용자 편집 시트는 절대 덮어쓰지 않음** — `sheet_exists()` 체크 후 분기
9. **참고컬럼(발표자 등)은 refresh, 배정컬럼(날짜 등)은 보존**
10. **특수 발표는 9000번대 합성 논문번호로 시트2에 주입**
